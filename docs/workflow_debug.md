## 3. 버그 수정 과정

### Step 4: 버그 수정 1차 시도 — 단순 순차 치환 (문제 있음)

```python
# 시도한 방식 (문제 있음)
if date_map:
    for old_mmdd, new_mmdd in date_map.items():
        new_val = new_val.replace(old_mmdd, new_mmdd)
```

### Step 5: 순서 충돌 문제 발견

단순 순차 치환은 다음과 같은 이중 치환 문제를 일으킨다.

```
date_map = {"06/18": "06/25", "06/25": "07/02", ...}
원본 문자열: "06/18(목) ~ 06/25(목)"

1차 치환 후: "06/25(목) ~ 06/25(목)"  ← "06/18" → "06/25"
2차 치환 후: "07/02(목) ~ 07/02(목)"  ← "06/25" → "07/02" (이중 치환 발생!)
```

치환 맵의 키와 값이 서로 교차하는 경우, 순차적 치환은 반드시 이 문제를 유발한다.

### Step 6: 단일 패스 regex로 최종 수정

`re.sub()`을 사용하면 이미 치환된 위치를 재방문하지 않으므로 이중 치환 문제가 해결된다.

```python
# 올바른 방식 — 단일 패스 regex 치환
_slash_pat = re.compile(
    "|".join(re.escape(k) for k in sorted(_slash_map, key=len, reverse=True))
)
new_val = _slash_pat.sub(lambda m: _slash_map[m.group(0)], new_val)
```

패턴을 키 길이 내림차순으로 정렬하는 이유: 더 긴 패턴이 짧은 패턴보다 먼저 매치되어야 오탐을 방지할 수 있다.

### Step 7: 점 형식 처리 추가

헤더 셀 `"06.18_ Event"`는 슬래시(`/`) 형식이 아닌 점(`.`) 형식을 사용하므로 별도 처리가 필요했다.

```python
# 점 형식용 파생 맵 생성
_dot_map = {k.replace("/", "."): v.replace("/", ".") for k, v in date_map.items()}
_dot_pat = re.compile(
    "|".join(re.escape(k) for k in sorted(_dot_map, key=len, reverse=True))
)
new_val = _dot_pat.sub(lambda m: _dot_map[m.group(0)], new_val)
```

## 6. 결론 및 교훈

### 교훈 1: 날짜 저장 형식 사전 파악 필수

xlsx 파일에서 날짜가 `datetime` 객체인지 문자열인지에 따라 치환 로직이 달라진다. 이 파일은 날짜가 전부 문자열로 저장되어 있었으며, 코드가 `datetime` 객체만 처리하도록 작성되어 있어 날짜 치환이 무시되었다. 파일 분석 단계에서 셀 타입을 먼저 확인하는 절차가 필요하다.

### 교훈 2: 단일 패스 치환 원칙

치환 맵의 키와 값이 서로 교차(예: A→B, B→C)하는 경우, 순차 `str.replace()` 방식은 반드시 이중 치환을 유발한다. `re.sub()`을 사용한 단일 패스 치환이 정답이며, 키를 길이 내림차순으로 정렬해 패턴 우선순위를 명시해야 한다.

### 교훈 3: 날짜 표기 형식 다양성 주의

동일 파일 내에서 `MM/DD`(슬래시)와 `MM.DD`(점) 두 가지 날짜 형식이 혼재할 수 있다. 헤더 셀과 본문 셀의 형식이 다를 수 있으므로, 치환 로직은 두 형식을 모두 커버해야 한다.

### 교훈 4: 자동화의 한계 — 경고 셀 수동 검토

이전 시즌이나 특정 이벤트를 참조하는 설명 문구는 규칙 기반 자동 치환이 불가능하다. 이런 셀은 경고 목록으로 출력하고, 담당자가 릴리즈 전에 직접 확인하는 프로세스가 필요하다.
